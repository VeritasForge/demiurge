# 대화 내용 구조화 정리

대화에서 논의한 내용을 **원문을 최대한 보존**하면서 **하나의 조화로운 문서**로 구조화하여 출력합니다.

```
Parse the user's input from: $ARGUMENTS
```

## Step 1: 대상 식별

- `$ARGUMENTS`가 있으면: 해당 주제에 해당하는 대화 내용만 추출
- `$ARGUMENTS`가 없으면: 대화에서 가장 최근에 깊이 논의된 주제 전체를 대상으로 함
- 대상이 되는 모든 assistant 응답과 user 질문을 파악

## Step 2: 원문 보존 원칙

다음 원칙을 **반드시** 준수:

1. **원문 유지**: 대화에서 사용된 설명, 비유, 코드 예시, 표를 최대한 원문 그대로 사용
2. **중복 병합**: 같은 내용이 반복된 경우 가장 완성도 높은 버전을 채택하되, 고유한 표현이나 비유는 보존
3. **정정 반영**: 잘못된 정보가 후속 대화에서 정정된 경우, **정정된 버전만** 반영 (틀린 정보는 포함하지 않음)
4. **맥락 보존**: 질문의 맥락이 답변 이해에 필요하면 자연스럽게 녹여넣음

## Step 3: 구조화 규칙

대화의 흐름을 분석하여 다음 구조로 재배치:

### 문서 구성
1. **제목(H1)**: 주제를 명확히 나타내는 제목 생성
2. **섹션(H2, H3)**: 논리적 흐름에 따라 구성
   - 관련 Q&A를 같은 섹션으로 묶음
   - 시간순이 아닌 **논리순**으로 배치
3. **배치 순서**: 배경 지식 → 핵심 설명 → 심화 내용 → 정리
4. **구분선**: 주요 섹션 사이에 `---` 사용

### 포맷팅
- **코드 블록**: 언어 식별자 포함 (```python, ```sql 등)
- **표**: 열 정렬된 마크다운 테이블 (헤더 구분선 필수)
- **비유/예시**: 원문에서 사용된 비유와 예시를 그대로 유지
- **핵심 정리**: 문서 마지막에 번호 목록으로 핵심 내용 요약

## Step 4: 출력

- 구조화된 마크다운 문서를 **직접 출력** (파일 저장 없음)
- 출력 전후에 부가 설명 없이 문서 내용만 출력
