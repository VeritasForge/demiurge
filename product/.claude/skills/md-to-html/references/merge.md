# Multi-File Merge Rules

`md-to-html a.md b.md c.md` 처럼 여러 마크다운을 **하나의 HTML**로 결합할 때의 규칙.

## 순서·구조

- **인자 순서 = 파트 순서**. a.md → b.md → c.md 순으로 배치.
- 각 파일 = **상위 "파트"**:
  - 파일의 첫 `# H1`(없으면 파일명 stem)이 **파트 타이틀**.
  - 파트 타이틀은 `.sec-title`급 헤더로, 파트 안의 원래 H2/H3는 한 단계씩 강등(파트 H1→H2, H2→H3 …)하여 **중복 H1을 만들지 않는다**.
- **단일 파일**이면 파트 그룹 없이 평면 구조 + 본문 첫 H1은 히어로 타이틀로 승격.

## 통합 TOC

- 사이드바 TOC를 **파트별 그룹**으로 구성:
  ```html
  <ol>
    <li><a href="#part-a"><span class="n">A</span>파트 A 제목</a></li>
    <li><a href="#part-a-sec1" style="padding-left:26px">하위 섹션</a></li>
    <li><a href="#part-b"><span class="n">B</span>파트 B 제목</a></li>
  </ol>
  ```
- 파트 경계는 `<section id="part-x">`로 구분하고, 파트 첫머리에 `.sec-head`로 파트 라벨(A/B/C…) 표시.

## Frontmatter 병합

- 각 파일 YAML frontmatter의 `tags` → **합집합(중복 제거)**, 히어로 태그칩으로.
- `created` 등 날짜 → 가장 이른~늦은 **범위** 또는 최신값. 히어로 메타에 표기.
- 제목 충돌: 결합 문서의 **히어로 타이틀**은 (a) 사용자가 지정했으면 그것, (b) 아니면 공통 주제를 요약한 한 줄 또는 첫 파일 제목 + "외 N건".

## 누락·에러

- 인자 파일 중 일부가 없으면: 존재하는 것만으로 진행하되 **누락 파일을 보고**하고 계속 여부를 물을지 판단(명백하면 진행 + 경고).
- 빈 파일/파싱 불가 → 건너뛰고 보고.

## 출력 파일명

- 다중 결합 시 기본 출력명: 사용자가 지정한 이름 > 없으면 `combined-<첫파일stem>-외<N>.html` 같이 명확히. 원본을 덮어쓰지 않는다(중복 시 `-2`, `-3` 증가).
