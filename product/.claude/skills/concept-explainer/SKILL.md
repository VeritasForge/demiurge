# Concept Explainer Skill

## Overview

이 스킬은 사용자가 질문한 **기술 개념**을 deep-research 프로토콜을 활용하여 조사한 뒤,
**8가지 표준 관점**으로 구조화된 리포트를 생성합니다.

ASCII 다이어그램, 테이블, 이모티콘을 적극 활용하여 **가독성 높은 시각적 리포트**를 제공합니다.

```
┌──────────────────────────────────────────────────┐
│          🔍 Concept Explainer Pipeline           │
├──────────────────────────────────────────────────┤
│                                                  │
│  Input: 사용자 질문 / 개념 / 기술 키워드          │
│    │                                             │
│    ▼                                             │
│  ┌────────────────────────────────┐              │
│  │ Phase A: Deep Research 수행    │              │
│  │ (deep-research 프로토콜 활용)   │              │
│  │ WebSearch + WebFetch + 교차검증 │              │
│  └──────────────┬─────────────────┘              │
│                  │                                │
│                  ▼                                │
│  ┌────────────────────────────────┐              │
│  │ Phase B: 8-Section 리포트 합성  │              │
│  │ 시각화 + 이모티콘 + 코드 예시   │              │
│  └──────────────┬─────────────────┘              │
│                  │                                │
│                  ▼                                │
│  Output: 구조화된 Concept Report                  │
│                                                  │
└──────────────────────────────────────────────────┘
```

## 실행 트리거 조건

이 스킬이 호출되면 **항상** 아래 프로토콜을 실행합니다.

**입력 형식**:
- `/concept-explainer <개념 또는 질문>`
- 예: `/concept-explainer Event Sourcing`
- 예: `/concept-explainer CQRS vs Event Sourcing`
- 예: `/concept-explainer gRPC`

비교 대상이 2개 이상이면 **섹션 6 (차이점)**을 특히 상세하게 작성합니다.

## Execution Instructions

### Phase A: Deep Research 수행

deep-research 프로토콜을 활용하여 개념에 대한 정보를 수집합니다.

#### Step A-1: 쿼리 분해

**mcp__sequential-thinking__sequentialthinking**을 사용하여:

1. 사용자 입력에서 **핵심 개념**과 **비교 대상**(있을 경우)을 추출
2. 8가지 관점 각각에 대한 **검색 쿼리 3-5개**를 영어로 도출
3. 검색 전략 수립

```yaml
query_plan:
  concept: "[핵심 개념]"
  comparison_targets: ["A", "B"]  # 비교 대상이 있는 경우
  search_queries:
    definition: "what is {concept} in software engineering"
    architecture: "{concept} architecture how it works internally"
    use_cases: "{concept} real world use cases best practices"
    pros_cons: "{concept} advantages disadvantages tradeoffs"
    comparison: "{concept_A} vs {concept_B} differences"
    pitfalls: "{concept} common mistakes anti-patterns"
```

#### Step A-2: 병렬 WebSearch (4-7회)

도출된 검색 쿼리로 **WebSearch**를 **병렬로** 실행합니다:

- 공식 문서, 기술 블로그, 커뮤니티(Reddit 등) 포함
- 최소 3개 이상 독립 출처 확보
- SNS 검색 1회 포함 (Reddit 우선: `site:reddit.com {keyword}`)

#### Step A-3: 심화 조사 (필요 시)

유망한 출처에 대해 **WebFetch**로 상세 내용 확인:
- 공식 문서의 아키텍처 다이어그램 설명
- 실무 적용 사례 (case study)
- 알려진 pitfall 및 anti-pattern

#### Step A-4: 교차 검증

수집된 정보를 교차 검증하여 신뢰도를 판정합니다.

---

### Phase B: 8-Section 리포트 합성

수집된 정보를 아래 **8가지 섹션**으로 구조화합니다.

**필수 포맷 규칙**:
1. 각 섹션 제목에 **번호와 이모티콘**을 포함
2. 핵심 내용은 **ASCII 테이블** 또는 **ASCII 다이어그램**으로 시각화
3. 중요 키워드는 **볼드** 처리
4. 코드가 관련된 경우 **코드 블록** 포함
5. 각 섹션은 **요약 → 상세** 순서로 작성
6. 출처는 인라인으로 `[출처명](URL)` 형식 포함

## 시각화 가이드라인

### ASCII 다이어그램 규칙

1. **박스**: `┌─┐ │ │ └─┘` 유니코드 박스 드로잉 문자 사용
2. **화살표**: `→ ← ↑ ↓ ▶ ◀ ▲ ▼` 방향 표시
3. **연결선**: `─ │ ├ ┤ ┬ ┴ ┼` 연결 표시
4. **흐름도**: 위에서 아래, 왼쪽에서 오른쪽 방향
5. **최소 너비**: 40자 이상으로 가독성 확보

```
# 예시: 레이어드 아키텍처
┌─────────────────────────────────┐
│        🖥️  Presentation         │
├─────────────────────────────────┤
│        ⚙️  Application          │
├─────────────────────────────────┤
│        🏗️  Domain               │
├─────────────────────────────────┤
│        💾  Infrastructure       │
└─────────────────────────────────┘
```

### 이모티콘 사용 규칙

| 용도 | 이모티콘 |
|------|---------|
| 섹션 번호 | 1️⃣ 2️⃣ 3️⃣ 4️⃣ 5️⃣ 6️⃣ 7️⃣ 8️⃣ |
| 장점 | ✅ |
| 단점 | ❌ |
| 경고/주의 | ⚠️ 🚫 |
| 팁/핵심 | 💡 📌 |
| 비교 | 📊 🔍 |
| 도구 | 🛠️ |
| 학습 | 📚 📖 📘 📺 🎓 |
| 트렌드 | 🔮 |
| 보안 | 🔒 |
| 성능 | ⚡ |
| 커뮤니티 | 💬 |
| 코드 | 💻 |
| 질문 | 🤔 |

### 테이블 규칙

1. 모든 테이블은 **헤더 구분선** 포함 (`| --- |`)
2. 셀 내용은 **간결하게** (긴 설명은 셀 밖에서)
3. 비교 테이블은 **3-5개 기준**으로 제한
4. 빈 셀은 `-` 또는 `N/A` 사용

## 품질 기준

### 필수 체크리스트

- [ ] 8개 섹션 모두 포함
- [ ] 각 섹션에 최소 1개 이상의 시각적 요소 (테이블, 다이어그램, 코드 블록)

- [ ] 이모티콘이 일관성 있게 사용됨

- [ ] 모든 주요 주장에 출처 포함
