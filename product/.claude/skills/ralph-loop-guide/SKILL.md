# Ralph Loop 사용 가이드

사용자에게 Ralph Loop 사용법을 안내합니다.

## 실행 절차

1. **프롬프트 템플릿 확인** - `~/.claude/ralph-loop-prompts/` 디렉토리의 템플릿을 읽어서 사용 가능한 템플릿 목록을 보여줍니다.

2. **가이드 제공** - 아래 내용을 기반으로 Ralph Loop 사용법을 안내합니다.

## Ralph Loop이란?

Geoffrey Huntley가 발명한 **자율 AI 개발 루프 기법**입니다.
Claude Code에 동일한 프롬프트를 반복 주입하여 작업이 완료될 때까지 자동 반복 실행합니다.

```
동작 원리:
프롬프트 주입 → Claude 작업 → 종료 시도 → Stop Hook 가로채기 → 같은 프롬프트 재주입 → 반복
```

각 반복에서 Claude는 이전에 자기가 수정한 파일과 git 히스토리를 읽고 점진적으로 개선합니다.

## 사용법

```bash
# 기본 (20회 반복)
/rl "프롬프트 내용"

# 반복 횟수 지정
/rl "프롬프트 내용" --max-iterations 30

# Ralph Loop 취소
/cancel-ralph
```

## 프롬프트 작성 Best Practice

### 1. 명확한 완료 기준 필수
```
완료 기준:
- [ ] 모든 테스트 통과
- [ ] 린터 에러 없음
- [ ] 구현 완료
모든 기준 충족 시 → <promise>COMPLETE</promise>
```

### 2. "하지 말 것" 섹션 필수
```
하지 말 것:
- 기존 테스트를 삭제하지 마
- 요청하지 않은 기능을 추가하지 마
- 범위 밖의 코드를 수정하지 마
```

### 3. 단계별 작업 프로세스
```
작업 프로세스:
1. 기존 코드 읽고 이해
2. 테스트 작성
3. 구현
4. 테스트 실행 → 실패 시 수정
5. 모든 기준 충족 → promise 출력
```

### 4. 2-Phase 워크플로우 (대규모 작업)
- **Phase 1**: 대화로 스펙/PRD 작성 → Markdown 파일 저장
- **Phase 2**: 새 세션에서 스펙 문서만 피드 → `/rl`로 자율 실행

## 안전 장치 체크리스트

```
[ ] --max-iterations 설정 (권장: 20~50)
[ ] 완료 기준이 객관적으로 검증 가능한가?
[ ] "하지 말 것" 섹션이 있는가?
[ ] git commit으로 복구 가능한 상태인가?
[ ] 첫 2~3번 반복을 직접 관찰할 수 있는가?
```

## 핵심 원칙

1. **반복 > 완벽성** — 첫 시도에 완벽할 필요 없음, 루프가 정제함
2. **실패는 데이터** — 실패를 관찰하고 프롬프트를 개선
3. **운영자 스킬이 핵심** — 좋은 모델 < 좋은 프롬프트 + 좋은 검증 전략
4. **끈기가 승리** — Eventual Consistency에 대한 믿음
