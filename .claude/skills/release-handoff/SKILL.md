# Release Handoff Reference Skill

## Overview

이 스킬은 릴리즈 핸드오프 문서의 **참조 지식**을 제공합니다. Release Investigator(`release-investigator.md`)가 문서 검증 시 이 스킬을 참조하여 완전성과 적절성을 판단합니다.

---

## 문서 구조 (7개 섹션)

| Section | 제목 | 핵심 내용 |
|---------|------|-----------|
| **1** | 릴리즈 개요 (Release Overview) | 버전, 날짜, 변경 등급, 요약, 담당자, 신규 모듈 여부 |
| **2** | 변경 사항 상세 (Change Details) | Breaking Changes, 신규 기능, 개선, 버그 수정, Deprecated/Removed |
| **3** | 아키텍처 변경 (Architecture Changes) | 컴포넌트 변경, 인프라 요구, 데이터 흐름, API 변경 |
| **4** | 배포 가이드 (Deployment Guide) | 사전 조건, 배포 절차, DB 마이그레이션, 환경 설정, 의존성 |
| **5** | 롤백 계획 (Rollback Plan) | 롤백 트리거, 절차, 데이터 복구, 사후 조치 |
| **6** | 배포 후 검증 (Post-Deployment Verification) | 스모크 테스트, 모니터링, 안정화 관찰 기간 |
| **7** | 신규 모듈/서비스 명세 (New Module Spec) | 모듈 개요, 아키텍처 위치, 인프라 요구, 의존성, 설정, DB, 운영 가이드 |

---

## 변경 등급 기준 (Change Level)

| 등급 | 기준 | 운영 영향 |
|------|------|-----------|
| **Major** | API 호환성 파괴, DB 스키마 변경, 아키텍처 재설계, 신규 모듈/서비스 추가 | 배포 전 충분한 검토 및 테스트 필요. 롤백 계획 필수 검토. |
| **Minor** | 신규 기능 추가, 설정 항목 추가 (하위 호환 유지) | 새 기능 관련 설정 확인 필요. 기존 기능 영향 없음. |
| **Patch** | 버그 수정, 보안 패치, 성능 개선 | 최소 영향. 표준 배포 절차 적용. |

## Hotfix 간소화 규칙

긴급 패치(Hotfix) 상황에서는 다음 섹션만으로 간소화할 수 있습니다:

| 필수 섹션 | 비고 |
|-----------|------|
| Section 1 (릴리즈 개요) | 변경 등급: Patch |
| Section 2-4 (버그 수정) | Hotfix 대상 이슈 |
| Section 4 (배포 가이드) | 간소화된 절차 |
| Section 5 (롤백 계획) | 최소한의 롤백 절차 |
| Section 6 (배포 후 검증) | 핵심 스모크 테스트만 |

**사후 보완 의무**: Hotfix 배포 후 72시간 이내에 전체 문서를 보완해야 합니다.

---

## 신규 모듈 포함 시 필수 규칙

릴리즈에 **완전히 새로 구현된 모듈 또는 서비스**가 포함된 경우:

1. Section 1의 "신규 모듈 포함 여부"에 **Yes** 표시
2. **Section 7 전체 작성 필수** (7-1 ~ 7-8 모든 하위 섹션)
3. 신규 모듈이 여러 개인 경우, 모듈별로 Section 7 반복 작성
4. Section 3 (아키텍처 변경)에서 신규 모듈의 시스템 내 위치 기술 필수

### Section 7 하위 섹션 (8개)

| 하위 섹션 | 내용 |
|-----------|------|
| **7-1** 모듈 개요 | 이름, 유형, 목적, 도입 배경, 담당자 |
| **7-2** 아키텍처 위치 | 시스템 내 위치, 연동 서비스, 데이터 흐름 |
| **7-3** 인프라 요구사항 | 런타임, 서버, 리소스, 네트워크, 방화벽 |
| **7-4** 의존성 | 외부 서비스, 라이브러리/패키지 |
| **7-5** 환경 설정 | 설정 파일, 환경 변수, Secrets |
| **7-6** 데이터베이스/저장소 | DB 종류, 스키마, 초기 데이터, 백업 정책 |
| **7-7** 설치 및 초기 구동 | Step-by-step 설치 절차 |
| **7-8** 운영 가이드 | 헬스체크, 로그, 모니터링, 스케일링, 장애 대응 |

## Edge Cases

| 상황 | 규칙 |
|------|------|
| **Hotfix** | 간소화 양식(Section 1, 2-4, 4, 5, 6만) 사용 가능. 사후 72시간 내 전체 보완. |
| **멀티 서비스 릴리즈** | 서비스별 Section 2-4 작성. Section 5(롤백)는 전체 릴리즈 단위로 관리. |
| **대용량 DB 마이그레이션** | 별도 실행 계획 및 리허설 필요. Section 4-3에 상세 기술. |
| **신규 모듈이 기존 모듈에 의존** | Section 7 설치 순서에 의존 관계 명시. 기존 모듈 버전 호환성 함께 기술. |
